Complexity Analysis of N-Queens

Time Complexity

The N-Queens algorithm uses backtracking to explore all possible solutions for placing N queens on an N x N chessboard. The time complexity can be analyzed as follows:

  • At each column, the algorithm tries to place a queen in each row, leading to a factorial number of recursive calls.
  • The number of recursive calls is given by: 
    T(N) = N * (N-1) * (N-2) * ... * 1 = O(N!)
  • Additionally, for each queen placement, validation of row, column, and diagonal conflicts takes O(N) time.
  • Thus, the total time complexity is: 
    T(N) = O(N!) * O(N) = O(N * N!)

Recurrence Relation

The recurrence relation for this problem is:

T(N) = N * T(N-1)

Expanding this, we get:

T(N) = N * (N-1) * (N-2) * ... * 1 = O(N!)

Space Complexity

The space complexity depends on two factors:

  • The board representation: 
    O(N^2)
    because the board is stored as a 2D array or a vector of strings.
  • The recursive call stack: 
    O(N)
    because the maximum depth of the recursion is equal to the number of columns.

Thus, the total space complexity is dominated by the board, resulting in:

O(N^2)

Conclusion

The complexities of the N-Queens algorithm are summarized as:

  • Time Complexity: O(N * N!)
  • Space Complexity: O(N^2)
